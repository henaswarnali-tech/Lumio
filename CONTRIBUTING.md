# Contributing to LUMIO

First — thank you. Every contribution to LUMIO, no matter how small, reaches a real child or adult who has no other access to education. That is not an exaggeration. It is the entire point.

This document explains how to contribute. You do not need to be a developer.

---

## Who Can Contribute?

Anyone. Seriously.

- You speak another language? You can translate lessons.
- You are a teacher? You can write or review curriculum content.
- You can draw? You can create illustrations or animations.
- You write code? You can build features or fix bugs.
- You have lived experience with ADHD, dyslexia, or another learning difference? Your feedback is more valuable than any developer's opinion.
- You know someone in an underserved community? You can test LUMIO with real users and report back.

---

## Ways To Contribute

### 📝 Curriculum Content
Write new lessons, review existing ones, or improve explanations.
- Lessons must be maximum 5 minutes to complete
- Use simple language — aim for clarity over sophistication
- Include at least one interactive element per lesson
- Follow the tone guide: warm, encouraging, never condescending
- Submit content as a markdown file in the `/curriculum` folder

### 🌍 Translations
Translate lessons or the app interface into any language.
- Create a folder in `/translations/[language-code]/`
- Translate one lesson at a time — small contributions are welcome
- If you are translating for a right-to-left language (Arabic, Urdu), note this in your pull request

### 🎨 Design and Illustrations
Create illustrations, animations, or UI improvements.
- Follow the design guidelines in `BRANDING.md`
- Character illustrations must represent global diversity
- Lottie animations must be under 50KB
- Submit design files with an explanation of what they are for

### 💻 Code Contributions
Fix bugs, build features, or improve performance.
- Read `SETUP.md` first to run the project locally
- One pull request per feature or fix
- Write a clear description of what you changed and why
- Test on both Android and iOS if possible

### 🧪 User Testing
Test LUMIO with real learners and report findings.
- Document who tested it (age, literacy level, location — no names)
- Note what confused them, what delighted them, what broke
- Submit findings as an issue with the label `user-testing`

---

## How To Submit a Contribution

1. **Fork** this repository (tap "Fork" at the top of this page)
2. **Make your changes** in your forked copy
3. **Submit a Pull Request** — tap "Contribute" then "Open Pull Request"
4. **Describe your changes** clearly in the pull request description
5. Wait for review — we aim to respond within 7 days

If you are new to GitHub and do not know how to do any of this, open an **Issue** instead and describe what you want to contribute. We will help you from there.

---

## Opening an Issue

Use issues to:
- Report a bug
- Suggest a new feature
- Ask a question
- Share user testing results
- Propose a new language or curriculum topic

Go to the **Issues** tab → tap **New Issue** → describe clearly what you found or want.

---

## Contribution Guidelines

**Be kind.** This project exists for vulnerable people. The community around it should reflect that.

**Be patient.** This is maintained by a very small team. We will respond, but not always immediately.

**Be honest.** If something does not work or a lesson is unclear, say so directly. Gentle honesty helps more than polite silence.

**No contribution is too small.** Fixing one spelling mistake in one lesson is a real contribution. It matters.

---

## Code of Conduct

LUMIO is a safe space for contributors of all backgrounds, nationalities, languages, neurotypes, and experience levels.

We do not tolerate:
- Discrimination of any kind
- Condescending or dismissive language
- Gatekeeping based on skill level or background

Anyone who violates this will be removed from the project.
